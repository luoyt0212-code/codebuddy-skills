## v4 核心改造

### ✅ 完整的导入流程

```
GetSeed API
    ↓
00 📒 Inbox（From_Get笔记）
    ↓

    ↓
主题分类文件夹（半导体/AI/商业航天/机器人/创投骆哥/发芽报告）
    ↓
[生成 4 个最终输出]
    ↓
本周索引 + 洞见雷达 + 知识卡片 + MOC 索引
```

### ✅ 四个最终输出

**1️⃣ 本周索引**（按日期查看笔记分类）
- 位置：`02 🗂️ 主题索引/本周索引/YYYY年第N周索引.md`
- 内容：
  - **Top 5**：从三星笔记中按评分挑选，配**AI智能总结的核心观点**（150-200字）
  - 按日期分组，显示星级、主题、评分
  - 笔记链接指向 `00 📒 Inbox（From_Get笔记）`
- **核心观点格式**：①主要论点/框架 ②关键洞察/反共识观点 ③actionable结论

**2️⃣ 洞见雷达**（文字描述 + 赛道统计）
- 位置：`02 🗂️ 主题索引/本周洞见/YYYY-WN-洞见雷达/`（每周一个新文件夹）
- 内容：
  - 价值分布统计（三星/二星/一星数量）
  - 赛道分布统计（每个赛道的星级分布）
  - 三星洞见详述（为什么有价值）
  - 二星洞见详述（为什么有价值）
- 笔记链接指向 `00 📒 Inbox（From_Get笔记）`

**3️⃣ 知识卡片**（Context + Insight + Connection）
- 位置：`06 🧠 Zettelkasten (卡片盒 - 核心洞察)`
- 来源：只从三星和二星笔记中提炼
- **命名协议（声明式标题）**：
  - ❌ 错误：`人工智能AI-AI分工.md`（标签式，看不出内容）
  - ✅ 正确：`人类负责搞定人，AI负责搞定事——AI时代的分工辩证法.md`（声明式，标题即结论）
  - **原则**：命名必须是一个完整命题，一眼看去就是结论，能直接嵌入新文章形成认知复利
- **原子化原则**：
  - 一张卡片只承载一个"变量"
  - 如果出现"而且"、"另外"、"此外"，拆分成两张卡片
  - 像乐高积木一样可自由组合
- **内容结构（C-I-C）**：
  - **Context（脱水背景）**：来源、触发场景、核心问题、关键数据
  - **Insight（洞见）**：一句话结论 + 三层论证 + 反共识观点
  - **Connection（关联钩子）**：向上连接（更抽象）、向下连接（具体应用）、横向连接（相关领域）
- **MOC 索引**：`02 🗂️ 主题索引/知识卡片索引/MOC - [主题].md`
  - 放弃深度嵌套文件夹，用 MOC 做组合管理
  - 卡片是底层资产，MOC 是持仓列表
- 逻辑：存量合并（追加到原卡片）或增量新建

### ✅ AI 四维度评分模型

| 维度 | 权重 | 说明 |
|------|------|------|
| 来源类型 | 40% | 发芽报告/项目决策 → 高分 |
| 原创度 | 30% | 包含个人思考关键词 → 高分 |
| 赛道相关性 | 20% | AI/机器人/航天等核心赛道 → 高分 |
| 可转化性 | 10% | 能否直接转化为内容/决策 → 高分 |

加权总分 ≥70 → 三星；≥50 → 二星；<50 → 一星

## 常见问题

**Q: 洞见雷达是表格还是文字？**
A: 是文字描述格式。显示价值分布统计、赛道分布、三星和二星洞见的具体描述。

**Q: 知识卡片的核心洞察要多长？**
A: 100-200 字。要拆解本质逻辑或非共识观点。

**Q: 如何调整原创度关键词库？**
A: 直接告诉小不要加什么词或删什么词，会自动更新脚本。

**Q: 笔记分类后能正常链接到 Inbox 吗？**
A: 可以。所有索引和洞见雷达中的笔记链接都指向 `00 📒 Inbox（From_Get笔记）` 中的笔记。

**Q: 如何重新执行某一周的导入？**
A: 使用 `python3 smart_import_to_obsidian.py --week N` 指定周次。
